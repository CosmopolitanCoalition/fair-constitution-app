### 1.2 Civic/electoral roles (chain)

R-01 Individual → R-02 Resident (GPS ping pattern) → R-03 Jurisdictionally Associated (point-in-polygon; **unlocks voting + candidacy with no other requirement**) → R-04 Voter (automatic) / R-05 Petitioner (self-initiated) / R-06 Candidate (self-registration) → R-07 Endorsed Candidate (org endorsement) / R-08 Election Board Member (legislative appointment). R-21 Advocate = R-03 + jurisdiction-law qualifications + judiciary exists.

## 4. Flow walkthroughs — actor → steps → entity transitions

## 5. Civic identity/residency lifecycle (consolidated)

```
Individual:       Registered → Identity-Verified → Residency-Declared → Resident (R-02)
                  → Jurisdictionally Associated (R-03) → [Relocating] → Re-associated | Deceased/Closed
Residency Claim:  Declared → Ping-Monitoring → Threshold-Met → Verified → Active
                  → Superseded (relocation) | Lapsed
```
Backend invariants the mockups encode: (a) identity verification is optional and never gates rights; (b) F-IND-006 auto-generates at threshold, user confirms or corrects (correct → re-declare F-IND-003); (c) verification triggers point-in-polygon resolution of the FULL ancestor chain — associations created at every nesting level simultaneously; (d) voting + candidacy unlock atomically at R-03 (hardened); (e) pings encrypted at rest, only derived day-count visible, purge on verification, pausable while traveling; (f) relocation requires the full CLK-05 threshold in the new jurisdiction before transfer — old claim becomes Superseded only when the new one Verifies (zero rights gap); (g) held offices get a grace period, then vacate into countback; (h) federation peers notified on transfer; (i) every civic action appends a hash-chained public-record entry (ballot participation recorded, ballot content never).

## 6. Full election lifecycle (consolidated)

```
Election:  Prior-Cycle-Certified → Approval-Phase-Open (continuous: registration + endorsements
           + revocable approvals) → Finalist-Cutoff (top X = f(seats), X pre-published, default 3×seats)
           → Ranked-Voting-Open (finalists + write-ins) → Voting-Closed → Tabulating (instant)
           → Certified → [Recount] → Final → next Approval-Phase-Open
```
Key mechanics as specified: two-phase open ballot (approval phase spans the entire inter-election period; only cutoff + ranked window are time-boxed); approvals secret individually / aggregate public daily; finalist line rendered live; non-finalists remain write-in eligible and write-ins are tabulated identically (right to stand preserved); ballots committed via commitment scheme (encryption + receipt hash + anonymized hash publication for self-audit); count = instant multi-winner STV, Droop quota `floor(valid ÷ (seats+1)) + 1`, Gregory fractional surplus transfers, exhausted ballots tracked per round, every seat per race fills in one count; full round-by-round record public with per-round transfer destinations; observer standing = endorsing organizations + candidates (no faction layer anywhere); certification (F-ELB-004) auto-converts winners to officeholders and re-opens the next approval phase; recount = in-system audit re-run, gated on certification + stated cause; vacancy → countback re-runs prior ballots minus vacated member (universal) → certify+seat+committee-proportionality-recheck, or exhausted → special election hard-windowed 90–180 days (engine rejects out-of-window dates); single-winner RCV exists ONLY for the individual-executive model with top-4 advisors by sequential exclusion; judicial elections = STV groups of ≥5; referendum questions ride the same ballot (F-IND-008) at majority/supermajority thresholds matching the originating act type, with CLK-19 protection on passage.
