Accessibility statement

Representation for every one means an interface every one can operate — across ability, language, script direction, device, and connection. This statement covers the mockup site itself and records the accessibility contract the production build inherits.

Target: WCAG 2.2 Level AA + selected AAA · EN 301 549 · last reviewed 2026-06-11

What is built in

  • Keyboard. Every interaction works without a pointer: a skip link is the first focusable element on every page, header menus are native disclosure widgets (Escape closes them), the ranked ballot uses move buttons rather than drag-only ordering, and flow walkthroughs are plain buttons and links. WCAG 2.1.1 · 2.1.2 · 2.4.1 · 2.5.7
  • Visible focus. A 3-pixel gold ring marks focus on every control; under Windows contrast themes the same outline renders in the system highlight color. Sticky chrome reserves scroll padding so the focused element is never hidden behind it. WCAG 2.4.7 · 2.4.11 · 2.4.13
  • Contrast. Text and interface colors are token-pairs checked against the AA ratios (4.5:1 text, 3:1 components); the quiet text tiers step up automatically under prefers-contrast: more. Color never carries meaning alone — every status pairs a label or icon with its hue, and the map layers use the Wong colorblind-safe palette. WCAG 1.4.1 · 1.4.3 · 1.4.11
  • Reflow. Layouts hold from 320 CSS pixels (and at 400% zoom) through phone, tablet, and large monitors, portrait or landscape, with no horizontal scrolling and no orientation lock. WCAG 1.3.4 · 1.4.4 · 1.4.10
  • Touch targets. Interactive elements are at least 24×24 CSS pixels including the compact chip controls. WCAG 2.5.8
  • Structure. Every page declares its language, carries a unique title, one h1, ordered heading levels, and the full landmark set (banner, navigation, main, contentinfo); status updates are announced through a persistent polite live region. WCAG 1.3.1 · 2.4.2 · 3.1.1 · 4.1.3
  • Motion. Animations are sub-300ms, nothing flashes, and prefers-reduced-motion disables all non-essential transitions. WCAG 2.3.1 · 2.3.3
  • No timeouts. Constitutional clocks on these screens are simulated displays — nothing on the mockup site expires or auto-advances. WCAG 2.2.1 · 2.2.3
  • Internationalization. All layout uses CSS logical properties so right-to-left scripts mirror correctly (flip the direction from the mockup controls to verify); the language switcher sits in the header on every page; a pseudo-locale with 35% text expansion is part of the QA toolchain. WCAG 3.1.1 · 3.1.2 · UAX #9
  • Offline-capable. Fonts are self-hosted (SIL OFL 1.1) and no resource loads from any external service — the site runs fully on a LAN with no internet access, on low-power hardware.

Known limitations

  • Page body copy ships in English; the locale switcher translates the application chrome only. The production build externalizes every string through an ICU MessageFormat catalog per the internationalization plan.
  • The stylized map panes (jurisdiction browser, district mapper) are placement stand-ins for the developed Leaflet viewers; each carries a text description, and the adjacent panel presents the same data as text. The production viewers ship with full keyboard navigation and text equivalents.
  • Automated and structural checks run on every page of this site; assistive-technology walkthroughs with human testers (screen reader, switch, voice control) are scheduled against the production build, not these static mockups.
  • The mockup controls bar at the foot of each screen is a demonstration instrument, not part of the application contract.

Feedback

If anything here is hard to perceive, operate, or understand, that is a defect in the design contract — report it to the Cosmopolitan Coalition so it is fixed before the production build inherits it.
